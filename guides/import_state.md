# Импорт ресурсов в состояние Terraform

Для импорта ресурсов в состояние Terraform можно использовать два метода: команду `terraform import` и блок `import` в конфигурации.

Команду `terraform import` — для того, чтобы импортировать ресурс в состояние terraform.

Используйте блок `import` в конфигурации, для того чтобы не только импортировать ресурс в состояние, но и заполнить поля конфигурации ресурса из состояния. Конфигурацию ресурса при этом можно сгенерировать.


## Метод 1: Использование команды `terraform import`
**Формат команды:**

```
terraform import <RESOURCE_ADDRESS> id=<RESOURCE_ID>
```

- `RESOURCE_ADDRESS` — адрес ресурса в конфигурации (включая модули, если применимо).
- `RESOURCE_ID` — идентификатор ресурса.

> **Важно:** Убедитесь, что в вашей конфигурации есть соответствующий ресурс с теми же полями схемы ресурса, что и импортируемый ресурс, иначе импорт может завершиться ошибкой.

**Пример (импорт подсети compute):**
```bash
terraform import module.my_compute_module.cloudru_evolution_compute_subnet.my_subnet 2babdcb8-57f6-42f4-894d-460d172e357e
```

---

## Метод 2: Использование блока `import`

Блок `import` позволяет описать импорт ресурса прямо в конфигурации Terraform, что делает процесс более декларативным и воспроизводимым.

Блок `import` работает только в версиях Terraform 1.5+. После выполнения `terraform apply` ресурс будет импортирован в состояние, после чего блок `import` можно удалить из конфигурации.

### Пример: Импорт подсети с помощью блока `import`

```hcl
import {
  id = "2babdcb8-57f6-42f4-894d-460d172e357e"
  to = cloudru_evolution_compute_subnet.my_subnet
}
```

Параметры блока `import`:

- `id` — идентификатор ресурса;
- `to` — целевой ресурс в конфигурации, в который необходимо импортировать состояние.  
  Укажите полный адрес ресурса, включая модули, если он находится внутри них.

> **Важно:** Убедитесь, что в вашей конфигурации есть соответствующий ресурс с теми же полями схемы ресурса, что и импортируемый ресурс, иначе импорт может завершиться ошибкой.

Если таргет-ресурс еще не определен в конфигурации, Terraform может автоматически сгенерировать его код:

```
terraform plan -generate-config-out=<PATH>
```

где `<PATH>` - путь к файлу, в который будет записан сгенерированный код ресурса. 
Команда сработает только если в конфигурации определен блок `import`. Имя сгенерированного ресурса будет соответствовать тому, что указано в `to` в блоке `import`.
