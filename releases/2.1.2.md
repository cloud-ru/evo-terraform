# Cloud.ru Evolution Terraform Provider - release 2.1.2
Важно!  
Версия 2.1.2 не поддерживает обратную совместимость с 2.1.2.  
Начиная с версии 2.1.0 провайдер поддерживает импорт (есть нюансы, о них ниже)

## Добавили
Добавили ресурсы:
- dataplatform/Spark
- dataplatform/Metastore
- dataplatform/Trino
- Artifact registry
- disk attach для compute (подключение/отключение дисков к VM)
- disk attach для baremetal (подключение/отключение дисков к серверам)
- subnet attach для baremetal  (подключение/отключение дисков к серверам)
- backup для postgres 
- kafka topic
- kafka user

Теперь провайдер поддерживает импорт. Но в некоторых ресурсах присутствуют параметры указываемые только при создании. При импорте, такие параметры не будут заполнены в манифесте. Это не мешает обновлять такие ресурсы, однако для пересоздания из потребуется прописать вручную. В будущих релизах это будет исправлено (обратная совместимость будет сохранена)

## Изменения
- Упростили структуру схемы для ресурсов (объединены некоторые поля, которые дублировались внутри схемы и вызывали некорректное поведение провайдера при импорте):
  - compute - disk, external_ip, image, interface, nat_gateway, security_group, security_group_rule, subnet, vm. 
  - mk8s - cluster, nodepool 
  - network - nlb, target_group
- Изменена работа с пользователями, теперь ФИО не управляется через тф (это исправляет баги управления пользователями)
